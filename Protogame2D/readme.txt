# Controls
Keyboard
| Key       | Action              |
|-----------|---------------------|
| S         | Turn left           |
| F         | Turn right          |
| E         | Thrust forward      |
| Space     | Fire bullet         |
| N         | Respawn ship        |
| I         | Spawn asteroid      |
| P         | Pause               |
| T (hold)  | Slow motion         |
| O         | Single frame step   |
| F1        | Toggle debug        |
| F8        | Restart             |
| Q         | Quit                |


Xbox controller
| Button / Stick     | Action          |
| ------------------ | --------------- |
| Left Analog Stick  | Move / Thrust   |
| Right Trigger (RT) | Fire bullet     |
| A                  | Shoot / Respawn |
| Start              | Pause / Begin   |

## Deep Learning:

At the start of this project I fell into some bad habits, most notably, not trusting or knowing how to use the debugger. 
Whenever I ran into an issue, I would reread my code, then immediately boot up the game to see if I could spot the error 
manually. This approach cost me hours that could have been saved by using breakpoints and other debugging tools. I’ve come 
to see the debugger as not just helpful, but essential.

I also realized the importance of planning before coding. A few times, especially in the math-heavy sections, I jumped 
into writing functions without fully understanding the problem. This led to confusion and time-consuming debugging. Simply 
sketching out the math or explaining the function out loud helped me catch mistakes before they happened. The lesson I’m 
taking forward is: don’t touch the keyboard until I can clearly articulate what I’m trying to do.

Lastly, I noticed my coding style was inconsistent. Variable names and function organization sometimes lacked clarity, 
which forced me to refactor midway through multiple times. To fix this, I've been trying to consciously review my naming and 
structure as I write, and I’ve asked for feedback from peers and the TA to improve code readability. These habits will 
help me write cleaner, more maintainable code moving forward.
